About FuelOil
    FuelOil is a two part Application Update Creator and Installer for Visual Basic 6. 
    Fuel is the Installer, Oil the update package creator.

What does Oil do?
Uses a complied executable to create an update installation package
Finds the executable's File name and version
Allows for entry of new files required for the update.
Allows for creation of new folders/directories.
Allows for new files to be .zip compressed
Allows for update to the uninstall log - ST6UNST.LOG, for VB's P&D wirard
Creates an install ini file that the Fuel install reads.
Copies any new files and the Fuel installer to the package folder.

What does the Fuel installer do?
Reads the install.oil (ini file)
Finds the Application to update's path
Finds the Windows System Dir.
Determines what files to install and where they go.
Installs the files, downloads then if needed.
Registers any System files (ocx,dll)
Updates the uninstall log - ST6UNST.LOG,  if selected to.

Fuel Installer

 Getting Started with Oil
   Before you begin using OIL you will need to compile the Fuel installer and place a copy in the App path of the Oil Project.
 

Creating Oil Files
    Oil will create a package of the Update list and Files into a folder for distribution.
Run Oil to begin creating the update package. When Oil creates an update package the package will consist of:

A folder with the name of your application followed by "Oil"  i.e.  MyAppOil
this folder will contain:
1. A text list of the files to be installed and where. (install.oil)
2. A folder named "Files" containing the files for updating.
    or a Zipped folder named "Files.zip" of you selected to compile the files.
3. A copy of the Fuel Installer.
4. A .gif image representing your App that will show up on the Fuel Installer.

 

Finding the App to Package
    When you run Oil the first thing you must do is determine is what app you are going to update.
Click the "Browse" button and locate the compiled app.
Oil reads the App's file information and gets the Name and version.
It is important the you package the newest version of your .exe.
Oil will remember any previous selection, which can be chosen from the combo box.

  If you created a setup/installation with VB's Package and Deployment Wizard,  Oil can tell the Fuel Installer to
update the uninstall log - ST6UNST.LOG for you new files.
  The EXE is NOT added to the uninstall log - ST6UNST.LOG when selecting this option, it already exists.


Where to install from
    You must  determine where the Update file are located.

Installing from the web
    Choose "Install from the web" if the update files are to be placed on a web sever.
You will be prompted for the URl of these files.  You will need to enter the full URL including subfolders.
You can upload the "Files" folder created by Oil to the server.  Make sure the URL ends with a "/".
When the Fuel Installer runs it will download and install the files from the URL.  Once The installation is complete Fuel will delete the downloaded files. You will not be able to zip the files if you chose this option. The Fuel installer doesn't have the ability to uncompress zip files at this time.

Installing form a File
    Choose "Install from a File" if the update files are to be put on a disk or downloaded as a Zip file.
The Fuel Installer will not delete the files after installation.  If "Installed from File" the "Files" folder created by Oil
must be in the same Directory or App.path as the Fuel Installer.

Do not change the "Files" folder to a different name.  The Fuel installer looks for the "Files" folder.

 

Creating Folders
    Once you have select where to install the files from you will be prompted for creating any new folders/directories for this update.
The new folders will be placed in your applications directory.  An example might be that you didn't include any help files with your original release and now you have some Html help pages that that need to be place in a "Help" folder inside your App's Directory.
Enter the folder names in to the textbox and click "Create New Folder".  If you need no new folders click "Next". Repeat as needed.

 

 Adding Files
     Clicking the "Add Files" button will open a dialog window in which you can select the files you need for this update, adding them to the List.  The EXE you selected to package should appear first in the list. Don't remove it, it would be bad. Continue until all the file need for this update are added.  As you add files you are prompted for their installation location by the "Where" window.
With your choices being:
AppPath: Your Application's Directory
 System: Window's System Folder
    Select this for system files such as ocx's and dll's.  Files with this install location will be registered by the Fuel installer.
New Folder: If you previously created any new folders they will appear as a option.

To remove a file, highlight it and uncheck the box.
Note:  It is important NOT to include any files that were part of an earlier installation or update.
Why?
1. You don't need to install them, they already exist.
2. The Fuel overwrites any existing files. 
3. If you selected to update the uninstall log. You could create duplicate entries, causing the App to uninstall improperly.
 

Add an Icon
    You may select to add a icon/image to the Fuel installer that represent your application.  The image must be in the form of a GIF and should have a transparent background.  The Image must not be larger than 55 x 55 pixels.  You do not have to select an image. 

 

Saving the Files
    Once you have added all the files that are required for the update.  Click the "Next" button.
Select where to save the OIL package. You can chose to have the "Files" folder compressed into a Zip format (Thanks to OldManMarcin). The Fuel installer and the Icon you selected will not be included in the Zip folder. The package will be saved in a folder with the name of your application followed by "Oil"  i.e.  MyAppOil

 

Fuel Installer
    The Fuel Installer will not run alone it needs an "install.oil" text file to read.  To test the Fuel installer, Run Oil and create a sample update for an existing Application.   It would be recommended to run VB's package and deployment  wizard and install your App before trying to update it.  If you run the App from the VB IDE and then run the Fuel Installer you may overwrite files you did not intend to.
